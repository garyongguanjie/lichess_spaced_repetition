About the Algorithm

← Back to puzzles

This extension uses the SM-2 spaced repetition algorithm (the same family of algorithms powering Anki) to schedule when you should revisit each saved Lichess puzzle.

How it works

Every puzzle is a "card" with three key fields:

  • Interval — how many days until the puzzle is shown again.
  • Ease — a multiplier (starts at 2.5) controlling how fast the interval grows.
  • Repetitions — how many times in a row you've passed it.

Grading

When you review a puzzle you mark it Pass or Fail:

  • Pass — the interval grows. The first pass sets it to 1 day, the second to 3 days, and afterwards the new interval is round(interval × ease). The ease increases slightly (+0.1) as a reward.
  • Fail — the interval resets to 1 day, repetitions return to 0, and the ease drops by 0.2 (never below 1.3) so future intervals grow more slowly.

Example schedule

RepsEaseNext interval
0 → 12.51 day
1 → 22.63 days
2 → 32.7~8 days
3 → 42.8~22 days

Why it works

Spaced repetition exploits the spacing effect: reviewing material just as you're about to forget it strengthens memory far more than cramming. By spacing puzzle reviews at increasing intervals, you spend less time on puzzles you know well and more on the ones you keep missing.
